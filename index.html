Procedimiento de Seguridad - Incidentes Web

Documento final elaborado individualmente

Objetivos

  • Desarrollar un procedimiento de seguridad (Resultado 3).
  • Aplicar legislación básica y buenas prácticas para la gestión de incidentes web.

Estrategias de Trabajo

  • Aprendizaje basado en problemas (ABP): el procedimiento se desarrolla a partir del análisis de incidentes reales.
  • Guía del facilitador: el docente orienta, supervisa y valida el proceso.
  • Exposición final: presentación individual del documento y análisis aplicado.

Tipos de Incidentes Web

  • Ataques por inyección SQL.
  • Cross-Site Scripting (XSS).
  • Denegación de servicio (DoS/DDoS).
  • Secuestro de sesiones (Session Hijacking).
  • Exposición de datos sensibles.
  • Manipulación de formularios o validaciones débiles.
  • Acceso no autorizado por contraseñas débiles.

Pasos para la Detección de Incidentes

  • Monitoreo continuo de tráfico y comportamiento del sistema.
  • Revisión de logs y alertas del servidor web.
  • Identificación de anomalías en el uso de recursos.
  • Notificación automática mediante sistemas IDS/IPS.
  • Verificación de actividades sospechosas del usuario.

Comunicación Interna

  • Informar inmediatamente al responsable de seguridad.
  • Registrar el incidente en el sistema interno.
  • Notificar al equipo técnico para análisis y contención.
  • Reportar afectaciones a la dirección o comités de TI.
  • Registrar acciones tomadas durante todo el proceso.

Contención, Erradicación y Recuperación

1. Contención

  • Aislar el sistema afectado.
  • Bloquear accesos externos sospechosos.
  • Desactivar servicios comprometidos temporalmente.

2. Erradicación

  • Eliminar malware, scripts maliciosos o archivos alterados.
  • Corregir configuraciones vulnerables.
  • Aplicar parches y actualizaciones pendientes.

3. Recuperación

  • Restaurar servicios afectados.
  • Verificar integridad de datos y sistemas.
  • Monitoreo posterior para asegurar estabilidad.

Recomendaciones Finales

  • Implementar políticas de seguridad basadas en ISO 27001 y NIST.
  • Realizar copias de seguridad periódicas.
  • Aplicar validación estricta en formularios web.
  • Configurar correctamente permisos de usuario.
  • Capacitar al personal sobre riesgos de seguridad.
  • Mantener actualizados todos los sistemas.
